
## 도커란?
- 컨테이너를 생성하고 관리하는 도구이자 컨테이너 기술
- 먼저 컨테이너가 무엇인지 알 필요가 있음

<br/>


## 컨테이너란?
- 컨테이너란 표준화된 소프트웨어 유닛, 코드 패키지이자 독립된 환경이며 각 어플리케이션에 필요한 종속성과 도구들이 포함되어 있다.
- 컨테이너는 단순히 독립적인 어플리케이션 구동만을 장점으로 하지는 것이 아니다. 컨테이너의 담긴 어플리케이션과 셋팅을 그대로 다른 pc, 클라우드 등 여러 곳에 공유 및 배포가 가능하다는 점이 큰 장점이라 할 수 있다.
- 우리는 코드 패키지와 코드를 실행하는 종속성을 보관할 수 있으며 도커가 실행되는 모든곳에 이를 그대로 가져올 수 있다. 따라서 어느곳에서든 동일한 환경에서 동일한 어플리케이션을 실행할 수 있다.
- 컨테이너는 대부분의 운영체제에서 내장되어 지원하거나 또는 도커를 설치하여 작업할 수 있기에 호환성이 뛰어나고 작업을 간단하게 만드는데 도움을 준다.


<br/>

## 왜 도커와 컨테이너를 사용하는가??
- 때때로 우리는 하나의 PC를 통해 다양한 프로젝트를 진행해야할 때가 존재한다. 그러나 로컬 pc 하나만으로 모든 프로젝트를 관리하기엔 하나의 라이브러리에 대해 프로젝트들이 요구하는 버젼이 다를 수도 있고 디피션시와 환경 등등 다양한 문제들이 발생할 것이다. 또한 로컬로 잘 작동하는 코드들이 다른 환경에서는 제대로 동작하지 않을수도 있다.
- 하지만 도커와 컨데이너를 사용한다면 애플리케이션이 자체적으로 필요한 버전을 제공하는 컨테이너에서 실행되기 때문에 도커와 컨테이너가 사용할 수 있는 곳이라면 어플리케이션의 실행 안정성과 재현성이 높아진다.

<br/>

## 왜 VM대신 도커와 컨테이너를 사용해야 하는가??
- VM의 경우 일종의 독립된 작은 컴퓨터라 할 수 있다. 프로그램에 필요한 모든 것과거기에 설치되는 모든 도구가 포함된 캡슐화된 버츄얼 머신이기 때문에 도커, 컨테이너와 동일한 결과를 얻을 수 있으나 컨테이너에 비해 크기나 용량등이 더 크므로 프로젝트별로 VM을 생성한다면 리소스의 소비가 비효율적이라 할 수 있다. 거기다 다른 곳에 배포를 해야할 때 VM을 만든다면 그만한 리소스 낭비가 없을 것이다.
- VM대신 컨테이너를 사용할 경우 VM만큼의 리소스가 낭비되지 않아 효율적이고 속도가 빠르며 도커 엔진을 통해 여러개의 컨테이너들을 한번에 관리가 가능하므로 공유, 재구축과 배포가 편리하다.
- 그렇다면 도커를 우리가 여러개의 프로젝트와 버젼별로 관리할 때 사용하는 가상환경과 비교한다면 어떨까??
- 나는 주로 아나콘다를 통해 여러 개의 환경을 구성하여 필요한 기능에 따라 바꿔가며 사용한다. 하지만 콘다를 이용한 가상환경은 로컬 PC에 종속되기 때문에 다른 PC에서 동일하게 사용할 수 없다. 하지만 도커의 컨테이너는 로컬 환경에 종속되지 않는다. 이미 컨테이너 안에 독립된 환경을 갖추고 있기 때문이다.

### 윈도우에 도커 설치하기
- 데스크탑 윈도우 버젼은 11 pro.
- 도커는 리눅스 기반이므로 윈도우 환경에서 도커 데스크탑만 설치한다면 WSL error가 뜨면서 제대로 동작하지 않는다.
- 따라서 윈도우 환경에서 리눅스를 사용할 수 있도록 도와주는 WSL2를 활성화하고 우분투를 설치하는 것이 필요하다.
- [윈도우 도커 설치방법](https://axce.tistory.com/121) 해당 글을 참고하여 WSL2를 활성화하고 우분투를 설치하는 것까지 진행하였다
- 그러나 우분투 설치에서 에러가 발생
  ```
  Installing, this may take a few minutes...
  WslRegisterDistribution failed with error: 0x80370102
  Please enable the Virtual Machine Platform Windows feature and ensure virtualization is enabled in the BIOS.
  ```

  바이오스에서 윈도우 가상환경 플랫폼과 가상화 기능이 설정되어야 한다는 에러가 발생함
     

- [WSL 에러 해결방법](https://gallery-k.tistory.com/311)
- [Windows에서 Linux 사용하기(with LG 노트북)](https://velog.io/@kleu358/Windows%EC%97%90%EC%84%9C-Linux-%EC%82%AC%EC%9A%A9%ED%95%98%EA%B8%B0with-LG-%EB%85%B8%ED%8A%B8%EB%B6%81)
- [WSL Error](https://velog.io/@jaylnne/WSL-Error-0x80370102-%ED%95%B4%EA%B2%B0)
- 위의 글들을 참고함.

  - 가상 머신 플랫폼 기능
  - Hyper-V 기능
    3가지 설정 후 재부팅시 F2 or Del 키로 바이오스 환경에 접속하기

- 하지만 블로그의 글들과 달리 바이오스 환경에 옛날 파란 배경이라는 문제가 있었음. 헤매다가 블로그에서는 Intel (VMX) Virtualization Technology라고 적혀있지만 내 컴퓨터 바이오스 화면에서는 인텔 가상화기술이라 적혀있는 부분을 활성화후 재부팅

  ```
  Installing, this may take a few minutes...

  ```
